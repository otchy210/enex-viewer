# ENEX Viewer 要件ガイド

フェーズをまたいだ要件整理を行うためのテンプレート集です。完了済みの Phase 1 の要件詳細は `docs/archive/phase1-requirements.md` に保存されています。

## 1. 目的と想定ユーザー

- プロダクトのゴールと代表的なユーザー像を明示する。
- レビュー時は過去フェーズとの差分（追加ユースケース/非機能要求など）をここに追記する。

## 2. スコープ定義テンプレート

```
### X.Y PhaseN に含める
- 箇条書きで機能を列挙

### X.Y PhaseN では含めない
- 将来へ見送る項目を列挙
```

必要に応じてフェーズごとに節を追加し、完了後は該当節を `docs/archive/phaseN-requirements.md` へ移動する。

## 3. 機能/非機能要件の記述ルール

- 機能要件: `FR-00x`、非機能要件: `NFR-00x` とし、テスト/ドキュメントの紐付けをコメントで残す。
- エージェントに依頼する際は、関連 FR/NFR をタスク概要に添えて追跡する。

## 4. 受け入れ条件サマリー

フェーズ完了時に満たすべき条件を箇条書きで記載し、具体的な検証結果は `docs/testing/manual-test-scenarios.md` とアーカイブ済み要件に残す。

## 5. アーカイブ参照

- Phase 1 要件詳細: `docs/archive/phase1-requirements.md`
- その他のフェーズも、完了時に `docs/archive/` 配下へ `phaseN-*.md` を追加し、このファイルには現行フェーズの方針のみを保持する。
