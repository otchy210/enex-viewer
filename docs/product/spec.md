# ENEX Viewer 仕様ガイド

フェーズごとの詳細仕様はフェーズ名付きドキュメントにアーカイブし、このファイルでは構成と更新ルールのみを維持する。Phase 1 の仕様全文は `docs/archive/phase1-spec.md` を参照。

## 1. ドキュメント構成テンプレート

1. ユースケース
2. 画面仕様（フェーズ名を見出しに含める）
3. API 仕様（OpenAPI への参照を必ず明記）
4. エラーハンドリング方針
5. Web 画面フロー
6. データ保持方針
7. 並列実装のための契約・モック方針

フェーズを追加したら、上記セクションを複製し、不要になった具体例は `docs/archive/phaseN-spec.md` へ移す。

## 2. 更新手順

- API/画面仕様を変更するタスクでは、同一 PR 内で `spec.md` を最新状態に更新する。
- 過去フェーズの仕様が最新と異なる場合は、当該セクションを `docs/archive/` へ移してから新内容を書き込む。

## 3. アーカイブ参照

- Phase 1 仕様: `docs/archive/phase1-spec.md`
- 以降のフェーズも、`phaseN-spec.md` の形式で追加し、このファイルからリンクする。
