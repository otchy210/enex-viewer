# 手動テストシナリオ（T-014）

## 1. 目的
ENEX Viewer の MVP に対して、人間が主要ユーザーフローを実操作で確認する。

## 2. 実施前提
- `nvm use` 実行済み
- `npm install` 実行済み
- `npm run dev` で API/Web が起動済み
- API: `http://localhost:3001`
- Web: `http://localhost:5173`

## 3. テストデータ
- `small.enex`: ノート数 10 程度
- `medium.enex`: ノート数 100 程度
- `broken.enex`: XML 破損データ

## 4. 実施シナリオ

### MT-001: 正常アップロード
- 目的: ENEX の取り込み開始と完了を確認
- 手順:
  1. Web を開く
  2. `small.enex` を選択してアップロード
- 期待結果:
  - エラーなく処理完了する
  - ノート一覧画面へ遷移または一覧が表示される

### MT-002: 一覧表示
- 目的: ノート一覧の主要情報表示を確認
- 手順:
  1. `small.enex` 取り込み後の一覧を確認
- 期待結果:
  - タイトル、作成日、更新日、タグ情報が表示される
  - 件数が取り込みノート数と整合する

### MT-003: 検索
- 目的: タイトル/本文/タグ検索の動作確認
- 手順:
  1. 一覧画面で既知キーワードを入力
  2. 結果件数を確認
  3. 該当しないキーワードを入力
- 期待結果:
  - 一致するノートのみ表示される
  - 非一致キーワードでは 0 件表示になる

### MT-004: ノート詳細表示
- 目的: 詳細情報と本文描画を確認
- 手順:
  1. 一覧から任意ノートを選択
- 期待結果:
  - タイトル、日時、タグ、本文、添付情報が表示される
  - 画面崩れがない

### MT-005: エラー表示（壊れた ENEX）
- 目的: 解析失敗時のユーザー向け表示確認
- 手順:
  1. `broken.enex` をアップロード
- 期待結果:
  - エラー内容が画面に表示される
  - アプリが操作不能にならない

### MT-006: 性能の目視確認（中規模 ENEX）
- 目的: 体感性能の確認
- 手順:
  1. `medium.enex` をアップロード
  2. 一覧表示、検索、詳細表示を操作
- 期待結果:
  - 主要操作が著しく遅延しない
  - ブラウザがフリーズしない

### MT-007: レスポンシブ表示
- 目的: 主要レイアウト崩れの有無確認
- 手順:
  1. ブラウザ開発者ツールでモバイル幅（例: 390x844）に切替
  2. アップロード、一覧、詳細を確認
- 期待結果:
  - 主要 UI が操作可能
  - 文字切れ・重なりが致命的でない
